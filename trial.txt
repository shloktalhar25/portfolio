# Serverless Crypto Data ELT Pipeline

## Project Overview

This project implements a **serverless ELT (Extract, Load, Transform) data pipeline** that ingests real-time cryptocurrency market data from the **CoinGecko API**, stores it in a **cloud data lake**, and integrates the data with **Snowflake** for analytics. The pipeline is designed using modern **cloud-native and Infrastructure-as-Code principles**, ensuring scalability, reproducibility, and automated deployment.

The system leverages **Azure Functions** for serverless ingestion, **Azure Data Lake Storage Gen2 (ADLS Gen2)** as the raw data lake, and **Snowflake External Stages** to enable the data warehouse to directly query files stored in the data lake.

---

# Problem Statement

Many analytics systems require ingesting **external API data at regular intervals** and storing it in a scalable and queryable format. However, building such pipelines manually can lead to inconsistent environments, deployment issues, and difficulty scaling.

This project demonstrates how to build a **cloud-native ELT pipeline** that:

* Automatically collects external API data
* Stores raw data in a scalable data lake
* Uses serverless infrastructure to reduce operational overhead
* Enables a data warehouse to access data lake files directly

# Architecture

Pipeline Architecture:

```
CoinGecko API
      ↓
Azure Function (Timer Trigger)
      ↓
Azure Data Lake Storage Gen2 (Raw Data)
      ↓
Snowflake External Stage
      ↓
Future Layer: dbt Transformations
      ↓
Future Layer: Analytics Dashboard
```

### Data Flow

1. **Azure Function** runs on a scheduled timer trigger.
2. The function fetches cryptocurrency market data from the **CoinGecko API**.
3. Raw API responses are stored as **JSON files** in **Azure Data Lake Storage Gen2**.
4. Data is partitioned using a **year/month/day folder structure** to improve scalability and query performance.
5. **Snowflake External Stage** is configured to read raw files directly from ADLS without duplicating storage.

# Key Implementation Details

### Serverless Data Ingestion

A **Python-based Azure Function** runs on a scheduled timer trigger to fetch cryptocurrency market data periodically from the CoinGecko API.

### Data Lake Storage Design

Raw API responses are stored in **Azure Data Lake Storage Gen2** using a **partitioned folder structure**:

```
raw/
   coingecko/
      year=2026/
         month=03/
            day=14/
               crypto_10_30.json
```

This structure enables efficient querying and scalable storage of time-series data.

### Infrastructure as Code

All cloud infrastructure resources—including:

* Azure Function App
* Storage Accounts
* Data Lake Containers
* Snowflake Database and Schema

were provisioned using **Terraform**, ensuring that the environment can be recreated consistently using a single command.

### Serverless Deployment Strategy

The Azure Function code was packaged using a **ZIP-based deployment method**, enabling immutable and repeatable deployments managed through Terraform.

### Data Warehouse Integration

A **Snowflake External Stage** was configured to allow Snowflake to read raw data files directly from the data lake, enabling future transformation workflows without duplicating data storage.
